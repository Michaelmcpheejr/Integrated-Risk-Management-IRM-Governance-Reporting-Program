# Control-to-Risk Mapping Framework

## Overview

The Control-to-Risk Mapping Framework establishes the relationship model between enterprise risks, controls, policies, issues, assets, and stakeholders.

The purpose of this framework is to:

- Improve governance traceability
- Strengthen audit readiness
- Centralize risk visibility
- Support remediation accountability
- Improve reporting consistency
- Demonstrate control coverage
- Align governance operations with ServiceNow IRM concepts

The framework ensures every risk can be traced to:

- impacted assets
- supporting controls
- applicable policies
- identified issues
- remediation activities
- accountable stakeholders
